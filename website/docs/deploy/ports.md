# 端口与安全组

## 云服务器安全组

必须开放：

```text
TCP 8090      面板访问
UDP 24642     Stardew 游戏端口
UDP 27015     查询端口
```

按需开放：

```text
TCP 5800      VNC/noVNC，仅在需要浏览器查看游戏画面时开放
```

不要开放：

```text
TCP 8080      Junimo API，仅供面板和容器网络内部访问，不需要公网开放
```

## NAS / 路由器端口转发

```text
局域网使用：
TCP 8090      面板访问
UDP 24642     Stardew 游戏端口
UDP 27015     查询端口

外网朋友加入：
在路由器上把 UDP 24642 / UDP 27015 转发到 NAS 的局域网 IP
如果需要外网管理面板，再转发 TCP 8090；不需要外网管理则不要转发 8090

按需：
TCP 5800      VNC/noVNC，仅需要浏览器看游戏画面时开放

不要开放：
TCP 8080      Junimo API，不要映射到公网
```

## 常见误区

- 最常见的"好友进不去游戏"问题是只开放了 TCP 端口，漏了 UDP：Stardew 游戏端口 `UDP 24642` 和查询端口 `UDP 27015` 都必须放行，且必须是 UDP 而不是 TCP。
- `TCP 8080`（Junimo 内部 API）不要开放到公网，它只供面板和容器网络内部通信。
- NAS 上不建议长期把 `TCP 8090` 转发到公网；更推荐局域网管理，外网管理使用 VPN、Tailscale、ZeroTier 或路由器自带 VPN。

更多故障排查见 [常见问题](/faq/) 中的"好友/局域网玩家进不去游戏"一节。
