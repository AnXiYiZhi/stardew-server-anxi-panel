# 面板管理与诊断

## 面板用户与权限

"设置"页（管理员可见）可以创建、禁用、删除面板用户：

- 管理员可以管理所有用户；普通用户只能登录、登出、查看自己的信息，看不到用户管理、Docker 状态等管理入口。
- 最后一个启用中的管理员账号不能被禁用、删除或降级，当前登录的管理员也不能禁用/删除自己，避免误操作导致无人能管理面板。
- 删除用户默认是软删除（禁用），需要显式选择才会硬删除。

## 日志与诊断

- "任务日志"页可以查看安装、启动、停止、备份等所有后台任务的实时日志和历史记录。
- "诊断"页可以查看 Docker/Compose 版本、健康检查详情，并导出诊断包（管理员可见的"导出诊断包"按钮），内含版本信息、健康检查、任务日志、Compose 状态等，已自动脱敏密码/Token/邀请码。
- 定期查看审计日志（管理员操作记录），确认没有非预期的账号或配置变更。

## 安全维护清单

- `PANEL_SECRET` 使用足够长的随机字符串，且不要对外泄露。
- 管理员账号使用强密码，定期检查是否有陌生管理员账号。
- 只放行必要端口：面板 `TCP 8090`，游戏 `UDP 24642`/`UDP 27015`，按需 VNC `TCP 5800`；不要把 Junimo 内部 API `TCP 8080` 暴露到公网。
- 挂载 Docker Socket 等于给了面板容器较高的宿主机 Docker 控制权，不建议在多人共用的宿主机上部署，也不建议把面板本身长期暴露在公网且不设强密码。
- NAS 用户如果需要在外网管理面板，优先用 VPN、Tailscale、ZeroTier 或路由器自带 VPN，而不是直接把 `TCP 8090` 转发到公网。
