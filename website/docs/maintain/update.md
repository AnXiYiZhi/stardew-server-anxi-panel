# 更新面板

## 一键脚本部署的服务器

```bash
cd ~ && bash run.sh update
```

如果更新后版本没变化（拉取失败或使用了旧缓存），强制重新拉取镜像并重建容器：

```bash
cd ~ && bash run.sh force-update
```

如果一键脚本本身也发布了新版本，先更新脚本再更新面板：

```bash
cd ~ && bash run.sh update-script
cd ~ && bash run.sh update
```

更新面板只会重建面板容器，不会删除 `~/.anxi-panel/data`；存档、Mod、数据库和备份都会继续保留。

## NAS 图形化部署的服务器

在项目/应用栈里重新拉取镜像并重启容器即可，同样不会影响挂载的数据目录。

## 更新后版本没变化怎么办

- 普通更新只会在本地已有对应镜像层时生效；如果拉取失败或使用了旧缓存，需要强制更新（见上方 `force-update`）。
- 如果一键脚本本身也有更新，先更新脚本再更新面板。
