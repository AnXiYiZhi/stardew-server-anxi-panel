# 更新面板

## 面板内直接升级（推荐）

管理员登录面板后，在"诊断"页可以看到当前面板版本和最新推荐版本。如果有新版本可用：

1. 点击"检查升级环境"会先做一次预检，确认面板所在容器支持升级。
2. 预检通过后点击"立即升级"，面板会在后台完成：下载新镜像、停止旧容器、用新镜像重建、验收健康状态。
3. 升级过程中面板会自动断开连接（因为容器本身正在重建），页面会显示"等待重连"并自动重试，升级完成后自动恢复。
4. 如果升级后新版本健康检查失败，面板会自动回滚到旧版本，整个过程不需要手动干预。

::: tip 升级前会自动备份
面板升级前会自动备份 SQLite 数据库，升级失败回滚时数据库也会同步恢复。存档、Mod 和游戏服务容器不在面板升级操作范围内，不会受到影响。
:::

::: warning 旧镜像自动清理
从 v0.3.7 起，升级成功后面板会自动清理本次被替换的旧镜像以及历史积累的过期 tag 镜像，释放磁盘空间。当前正在使用的镜像、其它项目的镜像和正在被其它容器引用的镜像会受到保护不会被误删。
:::

## 一键脚本部署的服务器

如果不方便通过面板升级，或面板本身健康检查失败无法正常工作，也可以通过命令行操作：

```bash
cd ~ && bash run.sh update
```

如果更新后版本没变化（拉取失败或使用了旧缓存），强制重新拉取镜像并重建容器：

```bash
cd ~ && bash run.sh force-update
```

如果一键脚本本身也发布了新版本，先更新脚本再更新面板：

```bash
cd ~ && bash run.sh update-script
cd ~ && bash run.sh update
```

## NAS 图形化部署的服务器

在项目/应用栈里重新拉取镜像并重启容器即可，同样不会影响挂载的数据目录。

## 更新后版本没变化怎么办

- 普通更新只会在本地已有对应镜像层时生效；如果拉取失败或使用了旧缓存，需要强制更新（见上方 `force-update`）。
- 如果一键脚本本身也有更新，先更新脚本再更新面板。
