# 任务与日志

面板里所有耗时操作（安装、启动、停止、重启、备份、Mod 安装等）都作为**后台任务（Job）**执行，不会卡住页面。"任务日志"页是查看这些任务实时进度和历史记录的地方。

## 任务列表

按时间倒序展示所有任务，每条包含：

- 展示名：优先显示对人类友好的名称，比如 Nexus/远程 Mod 安装会显示 `mod_nexus_install · Tractor Mod` 而不是单纯的任务类型代码。
- 状态：排队中、运行中、成功、失败。
- 触发方式：多数从对应功能页操作后自动跳转过来（比如安装页、Mod 页失败按钮），也可以直接在本页浏览历史任务。

## 实时日志

点开一个任务可以看到完整日志流，安装、Mod 下载等任务会显示进度信息（下载了多少 / 总量 / 百分比）。日志窗口内部滚动，不会带动整个页面跟着滚动。

## 常见排查思路

1. 某个操作失败后，先在"任务日志"里找到对应任务（通常刚失败的任务会排在最上面，或者从触发页面的失败提示直接跳转过来）。
2. 打开任务详情看完整日志，重点看最后几行的错误信息。
3. 网络类错误（下载失败、连接超时）通常可以直接重试；权限、配置类错误需要按提示修正后再重试。
4. 如果日志内容不足以定位问题，导出 [支持包](/handbook/diagnostics#支持包) 获取更完整的诊断信息。
