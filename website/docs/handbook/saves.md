# 存档管理

"存档"页管理服务器当前使用的存档、存档库里的其它存档，以及自动备份策略。

## 首次启动：新建或上传存档

服务器第一次启动、还没有任何存档时，面板会引导你二选一：

### 新建存档

在面板里直接生成一个全新农场，可配置：

- 农场名、玩家名（农民名）。
- 性别。
- 农场类型：**官方农场**包括标准、河边、森林、山顶、荒野、四角、海滩、草原农场共 8 种，每种都有缩略图预览；此外如果安装了支持自定义农场地图的 Mod（如 Stardew Valley Expanded 的"边境农场"），面板会自动扫描并以只读卡片形式展示已识别的模组农场，卡片上会标注当前依赖就绪状态（如缺少前置 Mod）。
- 初始宠物外观。

::: tip 模组农场目前只做展示
模组农场卡片目前为只读预览，不能直接选择创建。后续版本会在前置依赖和运行时验证都通过后开放创建入口。
:::

确认后面板会调用 JunimoServer 生成完整世界存档，这一步是同步阻塞的（要生成完整世界才返回），所以点击后需要等待一小段时间，不要重复点击。

### 上传存档

把自己电脑上的 Stardew Valley 存档打包成 ZIP 上传：

1. 拖拽 ZIP 到上传区，或点击选择文件。
2. 点击"预览存档"：面板会解析并展示存档里的农场名、游戏时间、地图类型、玩家名，供你确认这是不是想要的那份存档。
3. 确认无误后点击"导入并启动"完成导入。

上传导入支持来自其它平台（Steam/GOG 等）的存档：如果存档里记录的原平台主机玩家和当前服务器不匹配，面板会引导你选择原主机处理方式（替换平台 ID 或保留原身份），避免盲目接管导致数据丢失。

::: warning 上传/切换存档前必须先停服
存档相关的写操作（新建、切换、删除）都要求服务器当前不在运行/启动中状态，需要先停止服务器。
:::

## 切换 / 管理已有存档

存档库以卡片形式列出所有已导入的存档，每张卡可以：

- **选择**为当前启动存档。
- **导出**为 ZIP（备份到本地，或转移到其它地方）。
- **删除**（删除当前启动存档后，服务器会失去已选择的启动存档，下次启动前需要重新选择/新建/上传）。

## 删除离线玩家角色

服务器**运行中**时，管理员可以在"玩家"页对当前活动存档里的离线玩家角色执行删除：

- 只有满足以下条件的角色才允许删除：当前活动存档中、角色处于离线状态、不是主机角色、已经被玩家认领。
- 如果其他玩家在线，删除操作不会阻止他们继续游戏。
- 删除前面板会强制保存当前进度并创建一份整档保护备份，万一出问题可以随时恢复。
- 删除后该玩家重新上线时服务器会自动中止连接（提示该角色已不存在）。

::: warning 删除操作不可逆
角色被删除后存档里将永久移除该角色及其小屋，虽然面板会提前做保护备份，但仍然建议在操作前确认备份列表里有最近的存档备份。
:::

## 自动备份策略

左下角"自动备份策略"卡可以配置三种独立的自动备份方式，互不冲突可以同时开启：

| 策略 | 触发条件 | 生成的文件名 |
| --- | --- | --- |
| 游戏保存后自动更新最新备份 | 游戏内每次触发保存（含过夜自动保存） | `latest_<存档名>.zip`，会被覆盖更新 |
| 定时备份 | 每天指定的整点（0-23 时） | `scheduled_<存档名>.zip`，每天覆盖 |
| 每日快照 | 每天，额外按日期保留 | `daily_<存档名>_<日期>.zip`，滑块可调保留天数，默认 3 天，最长 14 天 |

## 备份列表：恢复与删除

备份列表里每个备份文件支持：

- **恢复**：用该备份覆盖当前存档前会弹窗二次确认，避免误操作丢失最新进度。
- **删除**：清理不需要的旧备份文件。

::: tip 什么时候该手动确认备份
批量安装/删除 Mod 之前、或者做任何可能影响存档的操作之前，建议先手动看一眼备份列表，确认最新一份备份是不是你期望的状态。
:::
