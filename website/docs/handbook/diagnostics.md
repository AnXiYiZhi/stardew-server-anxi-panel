# 诊断与支持包

"诊断"页用于排查面板/Docker 环境层面的问题，以及生成一份可以直接发给别人帮忙排障的支持包。

## 健康检查

进入诊断页会自动执行一次健康检查，包含 Docker 版本、Compose 版本等多项检查，每项显示通过/警告/错误状态。点击"重新检查"可以手动再跑一次。检查结果会同步显示回总览页的"系统健康"统计卡。

::: tip 为什么进总览页时健康卡是"未检查"
面板不会在你只是打开总览页时偷偷跑诊断命令，避免额外的后台开销。只有主动进过一次诊断页（或点了重新检查），总览页才会显示这次的结果。
:::

## 资源监控

诊断页展示 CPU、内存、磁盘三个圆环指标，实时反映容器资源占用。这个刷新只在诊断页打开且浏览器标签页可见时进行（每 8 秒一次），切到后台标签页或离开诊断页会自动停止轮询，避免不必要的资源开销。

## 支持包

管理员可见的"导出诊断包"按钮，会打包下载一个 ZIP，内含：

- 版本信息、健康检查结果
- 实例状态、任务列表、审计日志
- Docker Compose 状态、`docker-compose.yml`
- 服务器日志

::: tip 敏感信息已脱敏
支持包里的 Steam 密码、VNC 密码、Session、Token、邀请码等敏感信息都会被自动脱敏处理，可以放心把这个 ZIP 发给别人帮忙排查问题，不会泄露账号密码。
:::
