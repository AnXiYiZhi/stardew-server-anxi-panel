# Mod 管理

"模组"页是面板里功能最丰富的一页，分成三个工作台标签：**下载模组**、**添加模组**、**配置模组**。这篇文档按这三个标签逐一讲清楚每个按钮的作用，以及 Mod 从"装上"到"生效"再到"同步给玩家"的完整链路。

::: warning 改 Mod 前先停服
上传、删除、Nexus 一键安装等所有 Mod 写操作，都要求服务器当前**不在运行/启动中**状态。运行中这些按钮会被禁用并提示"服务器运行中，Mod 上传和删除已暂时禁用 - 请先停止服务器"。因为写入发生在停服状态，下次正常启动就会直接加载新 Mod——所以停服改完 Mod **不会**再提示"需要重启"；只有服务器运行中被系统标记为需要重启时（理论上不应发生，写操作本身已被禁用）才会看到"需要重启"提示。
:::

## 标签一：下载模组（从 Nexus 搜索安装）

### Nexus 连接状态条

标签顶部是一条状态提示，显示两类连接情况：

- **Nexus API Key**：管理员在"设置"页的"Nexus API Key"配置项里填入自己的 Nexus Mods API Key 后，这里会从"未配置/待配置"变成"已配置"。配置好 Key 后，Premium 用户搜索到 Mod 可以直接点"一键安装"走**直连下载**，不需要浏览器扩展。
- **扩展连接**：显示浏览器扩展是否已连接（未连接/检测中/已连接）。没有配置 API Key、或者不是 Nexus Premium 会员的情况下，一键安装依赖这个浏览器扩展来拿到真实的 CDN 下载链接。

管理员可以点"下载扩展"直接从面板拿到打包好的扩展 ZIP，解压后在浏览器扩展管理页（Chrome 的 `chrome://extensions`）以"加载已解压的扩展程序"方式加载。

::: tip Nexus 数字 ID 精确查询需要 Key
用 Mod 的数字 ID（而不是关键词）精确查询，需要先在"设置"页配置好管理员的 Nexus API Key；没配置时面板会明确提示缺少 Key，而不是报一个笼统的 500 错误。普通关键词搜索不需要配置 Key 就能用。
:::

### 搜索

搜索框支持输入英文模组名称、ID 或关键词；下方"热门标签"是几个预置的快捷搜索词（如 Tractor、Fishing Mod），点一下直接填入搜索框并触发搜索。不输入任何关键词直接点搜索，会显示近期热门 Nexus Mods。

### 搜索结果卡片

每张结果卡显示 Mod 名称、来源统计信息，以及：

- **前置依赖徽章**：没有前置依赖显示"前置：无"；有前置依赖时显示"前置已满足"或"缺少前置 mod"，点击可以展开查看具体缺哪个/哪些前置 Mod。
- **操作按钮**：跳转到 Nexus 官网查看详情，或直接"一键安装"。
- 点击"一键安装"后，按钮会变成进度条样式，实时显示下载百分比；如果失败，按钮会显示具体失败的 Mod 名（如"SpaceCore 失败"），点击可以跳转到 [任务与日志](/handbook/jobs-logs) 页查看失败原因。

::: tip 批量安装遇到重复怎么办
批量安装多个 Mod（含前置）时，如果某个 Mod 之前已经装过又被重复提交，面板会自动识别为"已安装，跳过重复导入"，不会算作失败中断整批安装。
:::

## 标签二：添加模组（手动上传 & 已装列表 & 玩家同步）

### 手动上传

支持一次选择/拖拽多个 ZIP 文件批量上传：

- 每个 Mod 文件夹里需要包含 `manifest.json`（标准 SMAPI Mod 结构）。
- Nexus 下载的 ZIP 如果外层多包了一层文件夹（常见打包习惯），面板会自动识别并剥离这层外壳，不需要你手动解压重新打包。
- 一次提交里如果某个 ZIP 导入失败，面板会回滚本次已经导入成功的其它 Mod，避免"上传 5 个只成功 2 个"的半成品状态。
- 重复上传已安装的相同 Mod（`UniqueID` 相同）会被明确识别为"已安装相同 ID 的 Mod"，不会被误判为 ZIP 损坏。

### 统计条

顶部四个统计数字：已安装数量（不含 SMAPI、面板控制组件等系统运行组件）、服务器状态、"运行中重启"标记（是否有历史遗留的重启提示）、解析失败数量（manifest 解析失败的 Mod 会单独计数并在列表里标红）。

### 玩家同步

多人联机时，Mod 分三类同步归属，可以在"配置模组"标签或已安装卡片上手动指定：

| 分类 | 含义 |
| --- | --- |
| 服务器专用 | 只需要服务器端有，玩家不需要装同款 Mod |
| 玩家需同步 | 每个进服玩家的客户端也必须装这个 Mod，否则会报错或不同步 |
| 待确认 | 还没手动分类，面板不确定，建议尽快确认 |

面板提供两个导出按钮，生成给玩家用的自解压安装包：

- **导出完整同步包**：给第一次加入的玩家用，里面包含 SMAPI 安装器 + 所有"玩家需同步"的 Mod。
- **导出模组更新包**：给已经装过一次同步包的老玩家用，不含 SMAPI，只更新 Mod。

玩家拿到 ZIP 后运行里面的 `安装玩家同步包.bat`（Windows）即可自动完成安装：脚本会校验文件完整性、按内容指纹跳过完全相同的 Mod（不用重复覆盖）、备份并替换内容不同的同名 Mod、安装 SMAPI（如果需要），并尽量帮玩家写好 Steam 启动项。卸载用同包里的 `卸载本同步包.bat`。

### 已安装列表

每张已装 Mod 卡片显示启用/禁用状态、同步分类标签、前置依赖状态；非内置 Mod 可以点"删除"移除（如果和其它 Mod 同属一个包，会提示会一起删除几个）。SMAPI、面板控制组件（`StardewAnxiPanel.Control`）、官方 `JunimoServer` 组件这些系统内置项会显示"内置"标记，不能删除。

## 标签三：配置模组（按存档启用/禁用）

Mod 的启用状态是**按存档保存**的，需要先在其它页面选好当前存档才能操作这里：

- 每个用户可见 Mod 一行，显示 `UniqueID`、前置依赖标签、启用/禁用开关。
- 切换开关会联动同一个包内的其它 Mod 和前置 Mod（比如禁用一个前置 Mod 依赖的主 Mod 时会一并处理）。
- 如果某个 Mod 因为缺少前置而不能切换，开关会被禁用并提示原因。

## 完整操作顺序参考

1. 停止服务器。
2. 到"下载模组"标签搜索并一键安装，或到"添加模组"标签手动上传 ZIP。
3. 到"配置模组"标签确认新 Mod 已启用（针对当前存档）。
4. 如果是需要玩家客户端同步的 Mod，到"添加模组"标签把它标记为"玩家需同步"，导出对应的同步包发给玩家。
5. 启动服务器验证 Mod 正常加载。
