# 安装游戏

"安装"页负责把 Stardew Valley 专用服务器（JunimoServer）装进面板容器管理的 Docker 环境。这是唯一需要你提供 Steam 账号信息的地方。

## 首次安装

1. 填写 **Steam 账号**、**Steam 密码**、**VNC 密码**（VNC 密码用于后续通过浏览器画面查看游戏窗口，不是 Steam 密码）。
2. 点击开始安装。面板会依次执行：
   - 拉取 JunimoServer 相关 Docker 镜像。
   - 用你的 Steam 账号登录官方 `steam-auth` 服务。
   - 下载 Stardew Valley 游戏文件（Steam App 413150）。
   - 下载 Steamworks SDK（用于服务器内部渲染/邀请码等能力）。
   - 安装 SMAPI 运行环境（Mod 加载器，见 [Mod 管理](/handbook/mods)）。
3. 全部完成后回到"总览"或"服务器"页启动服务器。

整个过程页面会实时展示当前处于哪一步，以及每一步的下载进度。

## Steam Guard 验证

如果 Steam 账号开启了两步验证，安装过程中会暂停并提示以下几种方式之一（具体哪种取决于账号的验证方式）：

- **扫码登录**：显示二维码，用手机 Steam App 扫码确认。
- **手机 App 批准**：直接在手机 Steam App 上点确认，不需要输入验证码。
- **验证码输入**：邮箱或 Steam 令牌发来的验证码，填入面板提供的输入框提交。

提交验证码后页面会先显示"验证码已提交，正在等待 Steam 响应"，不会立刻消失或报错；如果长时间没反应，页面提供"重新输入"按钮。

::: tip 为什么要输两次验证码
游戏本体下载和 Steamworks SDK 下载走的是两个独立登录会话；SDK 下载固定用匿名登录，不需要账号，所以正常情况下一次安装最多批准一次 Steam Guard，不会反复弹验证码。
:::

## 国内网络：SteamCMD 自动兜底

如果 `steam-auth` 认证成功但游戏文件下载失败（常见于国内网络访问 Steam CDN 不稳定），面板会在**同一个安装任务里自动切换到 SteamCMD** 重新下载，复用你已经填过的账号密码，不需要重新操作。安装页会明确提示"已自动改用 SteamCMD 下载"，并继续展示下载进度。

如果 SteamCMD 本身也需要重新验证，页面会展示对应的验证码/手机批准提示，处理方式和上面一致。

## 修复 / 重新安装

已安装过的实例，安装页会显示"重新安装 / 修复"入口，用于游戏文件损坏、下载中断等情况：

- 点击后**不需要**重新输入 Steam 账号密码，面板直接复用已保存的凭据和授权缓存，跳过 `steam-auth` 登录环节，直达文件下载/校验。
- 已下载的游戏文件不会被清空重来，SteamCMD 会校验已有文件，只重新下载有问题的部分。

## 更换 Steam 账号 / 强制重新认证

如果要换一个 Steam 账号建服（例如原账号被封禁、或者想用另一个账号的 Stardew 授权），用"更换 Steam 账号 / 重新认证"入口：

- 需要重新输入新账号的用户名、密码和 VNC 密码。
- 面板会清除本地保存的 Steam / SteamCMD 授权缓存，用新账号走一遍完整认证流程。
- 已下载的游戏文件（`game-data`）会保留，不用重新下载几个 G 的游戏本体。

## 安装失败常见现象

| 现象 | 说明 |
| --- | --- |
| 镜像拉取失败 | 面板会在复用凭据重试时自动重新拉镜像并重走认证，不会误跳到 SteamCMD |
| SteamCMD 授权缓存不可用 | 需要重新走一遍账号密码 + Steam Guard |
| SMAPI 安装失败 | 游戏文件和 SDK 都已下载完成，只是 SMAPI 运行环境安装失败；可以直接重试这一步，不影响已通过的 Steam 认证 |

更多报错现象和解决办法看 [常见问题](/faq/)。
