# 面板设置

"设置"页管理员可见，集中了面板本身的信息、端口配置、用户管理、审计日志和安全状态。普通用户看不到这些入口，参考 [账号与权限](/handbook/accounts)。

## 面板版本

显示当前运行的面板镜像版本，方便对照 [更新面板](/maintain/update) 确认是否已经是最新版本。

## 端口信息

显示并可修改面板端口、VNC 端口（供浏览器 noVNC 查看游戏画面用）。

::: warning 改端口后记得同步防火墙/安全组
改完端口只是让面板容器监听新端口，云服务器安全组或路由器端口转发规则需要你自己同步修改，否则改完反而连不上。完整端口清单看 [端口与安全组](/deploy/ports)。
:::

## 用户管理

创建、禁用、启用、删除面板用户，角色区分管理员/普通用户，详细规则和限制看 [账号与权限](/handbook/accounts#用户管理)。

## 审计日志

记录管理员的敏感操作历史（创建/删除用户、修改设置等），建议定期查看确认没有非预期变更。

## 安全与权限概览

只读展示当前的安全基线状态，正常情况下应全部显示"已启用/安全"：

| 项目 | 说明 |
| --- | --- |
| Session 认证 | 管理员会话认证已启用 |
| 密码哈希 | 使用 bcrypt 强哈希，不存明文密码 |
| 操作审计 | 记录敏感操作日志 |
| Docker Socket | 未挂载到容器内部（面板容器本身不能通过挂载点直接操控宿主 Docker） |
| 日志脱敏 | 已屏蔽敏感信息输出 |

## 安全维护清单

以下是需要你自己动手确认的安全事项，面板不会自动帮你检查：

- `PANEL_SECRET` 使用足够长的随机字符串，且不要对外泄露。
- 管理员账号使用强密码，定期检查是否有陌生管理员账号。
- 只放行必要端口：面板 `TCP 8090`，游戏 `UDP 24642`/`UDP 27015`，按需 `TCP 5800`（VNC）；**不要**把 Junimo 内部 API `TCP 8080` 暴露到公网。
- 挂载 Docker Socket 等于给了面板容器较高的宿主机 Docker 控制权，不建议在多人共用的宿主机上部署，也不建议把面板本身长期暴露在公网且不设强密码。
- NAS 用户如果需要在外网管理面板，优先用 VPN、Tailscale、ZeroTier 或路由器自带 VPN，而不是直接把 `TCP 8090` 转发到公网。

::: tip 界面主题、多游戏模式等"其他设置"
设置页底部还有界面主题、界面语言、多游戏模式、通知设置等条目，目前标注"后端待接入"，属于规划中但尚未实现的功能，不要按这些条目现在能生效来配置。
:::
