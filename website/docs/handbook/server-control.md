# 服务器控制

"服务器"页（总览页也有一份精简版）集中管理服务器的启停、邀请玩家和运行时控制台操作。

## 生命周期按钮

启动 / 停止 / 重启三个按钮，按钮是否可点由服务器当前状态决定：

- 已停止时才能"启动"，运行中才能"停止"或"重启"。
- 有后台安装/生命周期任务正在跑时按钮会显示对应的过渡态（启动中…/停止中…），避免重复点击。
- 启动不会等待邀请码拿到才算成功——面板会先把容器拉起来，再在后台轮询获取邀请码，邀请码稍后自动出现在邀请码框里，不会卡住"启动"按钮。

## 邀请码与联机方式

启动成功后，"邀请码"框会显示 Steam 邀请码，直接分享给好友，好友通过 Steam 好友邀请或输入邀请码即可加入。

下方"局域网邀请"框显示的是**你当前浏览器访问面板所用的地址**（不是探测出来的公网 IP），方便同一局域网内的人直接用这个地址+游戏端口连接；如果你是用公网域名/IP 访问面板的，这里显示的也会是对应的公网地址。

::: tip IP 直连默认开启
面板会在启动前自动确保存档设置里的 `AllowIpConnections` 为开启状态，作为邀请码走 Steam 网络失败时的可靠兜底连接方式。云服务器要让好友能 IP 直连，需要在安全组放行 `UDP 24642`，见 [端口与安全组](/deploy/ports)。
:::

## 计划重启（定时维护窗口）

"计划重启"卡片可以设置每天固定的关闭时间和开启时间，用于定期重启服务器释放内存、应用配置变更等：

- 可设置关闭前提前 N 分钟，通过游戏内聊天广播提醒在线玩家。
- 可勾选"关闭前自动备份当前存档"，关闭前先打包一份存档备份。
- 可勾选"有玩家在线时跳过本次关闭"，避免打断正在游玩的玩家。
- 到设定开启时间会自动重新启动；如果面板本身重启过或离线了一段时间，只在开启时间之后 5 分钟的宽限窗口内补跑一次，不会把错过很久的时间点也补上。

总览页右侧信息栏会实时显示当前处于"等待关机 / 关机中 / 等待开机 / 开机中"的哪个阶段和倒计时。

## 服务器密码设置

"服务器密码设置"按钮（管理员可见）用于配置玩家加入服务器所需的密码：

- 弹窗内输入密码后点"保存"，密码会写入服务器配置；**必须重启服务器容器后才会生效**，因为 JunimoServer 只在容器启动时读取一次密码，不支持运行中热改。留空表示不设置密码。
- 玩家加入后需要在游戏内聊天框输入 `!login 密码` 完成认证，认证成功前无法在地图上自由活动。
- 弹窗下半部分"密码保护状态"展示当前是否启用了密码保护、已认证/待认证玩家数量、认证超时秒数和最大失败次数，服务器运行中才能读取，可点"刷新"重新拉取。

## 全服喊话

在服务器运行中，"全服喊话"框可以输入一段文字（最多 200 字符）广播到游戏内聊天，不需要进游戏本身操作。

## 控制台命令

"控制台命令"下拉框提供一组**后端 allowlist 白名单命令**，不是任意 shell 命令：

| 命令 | 说明 | 是否仅管理员 |
| --- | --- | --- |
| 服务器信息 | 查看当前状态、玩家数、存档信息 | 否 |
| 邀请码 | 获取当前邀请码 | 否 |
| 查看设置 | 显示当前服务器设置 | 是 |
| 校验设置 | 校验服务器设置是否有效 | 是 |
| 渲染状态 | 查看服务器渲染状态 | 是 |
| 自动托管状态 | 查看 host-auto 设置 | 是 |
| 可见性状态 | 查看 host-visibility 设置 | 是 |

选中命令后点击执行，输出会显示在下方终端样式的结果框里。

::: warning 为什么不能输入任意命令
面板不在前端暴露任意 shell 或任意 SMAPI 命令，只允许执行这份固定白名单，避免误操作或恶意命令影响服务器稳定性和安全。
:::
