# 界面总览

面板登录后进入的是一个像素风格的"星露谷主机 Shell"，布局固定分三块：顶栏、左侧（或窄屏下顶部）导航、中间内容区，部分页面右侧还有信息栏。

## 顶栏

从左到右依次是：

- **品牌标题**：`Stardew Anxi Panel`，纯展示。
- **状态牌**：显示服务器当前状态——`运行中` / `已停止` / 启动中 / 读取中等过渡态。运行中和已停止用不同颜色的像素牌面区分，一眼能看出服务器是否在跑。
- **存档框**：显示当前启动存档的农场名和简略游戏时间，格式类似 `AnxiFarm：第一年春`（只到年份+季节，不显示具体日期）。没有存档时这里为空。
- **用户框**：显示当前登录用户的头像、用户名和在线绿点，点击进入 [设置页](/handbook/settings)。

## 左侧 / 顶部导航

宽屏下是左侧纵向导航栏，窄屏（手机、竖屏平板）下自动折叠成顶部横向选择栏。9 个入口固定顺序：

| 图标顺序 | 页面 | 对应文档 |
| --- | --- | --- |
| 1 | 总览 | 本页下方"总览页" |
| 2 | 服务器 | [服务器控制](/handbook/server-control) |
| 3 | 存档 | [存档管理](/handbook/saves) |
| 4 | 任务日志 | [任务与日志](/handbook/jobs-logs) |
| 5 | 玩家 | [玩家管理](/handbook/players) |
| 6 | 模组 | [Mod 管理](/handbook/mods) |
| 7 | 诊断 | [诊断与支持包](/handbook/diagnostics) |
| 8 | 安装 | [安装游戏](/handbook/install) |
| 9 | 设置 | [面板设置](/handbook/settings) |

## 总览页

进面板后的默认落地页，是其它页面的信息汇总，不单独产生新操作：

- **四张统计卡**：存档数量、模组数量（不含 SMAPI/面板内置组件等系统运行组件）、系统健康（评分/通过项，需要先进过一次[诊断页](/handbook/diagnostics)才会有数值，否则显示"未检查"）、运行中任务数量。
- **服务器控制卡**：和"服务器"页共用同一套启动/停止/重启按钮，以及邀请码、局域网邀请（当前面板访问地址）两个信息框。
- **右侧信息栏（OpsRail）**：显示当前"进行中"的后台任务，以及如果配置了计划重启，会显示到下次自动关机/开机的倒计时和当前所处阶段（等待关机、关机中、等待开机、开机中）。

::: tip 为什么系统健康卡一直显示"未检查"
面板默认不会在你进入总览页时偷偷跑 Docker 版本检查等诊断命令，避免不必要的后台开销。只有你主动打开一次"诊断"页，或点击诊断页的"重新检查"，总览页的健康卡才会显示评分。
:::
