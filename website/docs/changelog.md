# 版本更新日志

Anxi Panel 的正式版本号对应 Docker Hub 上发布的镜像 tag。以下是主要版本的用户可见变更摘要。

::: tip GitHub Release
每个版本都对应一个 GitHub Release，可以在 [GitHub Releases](https://github.com/AnXiYiZhi/stardew-server-anxi-panel/releases) 查看每版的完整变更说明。
:::

## v0.4.1（最新版本）

**飞牛 OS 迁移脚本更新**：新版迁移脚本可识别并保留额外 bind mount 与具名 Docker volume，迁移成功前会复核 Compose、labels、容器、镜像 digest 和数据挂载是否一致；修复飞牛残留 Compose labels 导致旧容器被误认为新服务的问题，并在确认支持后续 Web 一键升级时输出固定成功识别码。文档首页同步优化快速上手按钮和版本更新卡片的显示效果。

## v0.4.0

**一键全栈安全升级**：Panel 现在可以安全识别自定义 Compose 服务名，并把符合安全边界的飞牛旧容器转换为标准 Compose 部署；升级前自动保护数据库、部署配置、环境变量和旧镜像摘要，新 Panel 健康失败会恢复旧容器。Panel 更新后自动逐实例核对 Control 版本与 DLL hash，必要时依次通告在线玩家、保存、创建整档备份、重启游戏并验证 SMAPI 实际加载版本；失败实例保持停止，升级状态可在 Panel 重启后继续。

## v0.3.13

**存档上传编码与删除一致性修复**：修复中文 Windows 生成的 GBK/GB18030 编码 ZIP 存档在上传预览时名称显示为乱码的问题，上传前自动规范为 UTF-8；历史非 UTF-8 目录以稳定公开身份列出，正常中文可读，无法识别或同名冲突使用哈希别名；修复删除事务中"目录已删但指针清理报失败"的假失败问题，重复删除正确返回 404。

## v0.3.12

**低资源主机升级冷启动修复**：修复低资源主机（如 NAS、小内存云服务器）升级后因冷启动超时导致误回滚的问题，server 升级验收默认延长至 20 分钟，server/auth stop 失败在 10 分钟内按同一 allowlist 幂等重试；新 Compose 与旧实例迁移增加 steam-auth/server CPU shares 优先级分配。

## v0.3.11

**删除人物后无人暂停修复**：修复删除离线玩家角色后服务器因 Control Mod 0.2.1 的暂停策略反馈锁导致时间冻结的问题，Control 0.2.2 简化暂停策略，只接收连接数、节日和时间边界，删除 gameplay player/menu request 计数，菜单暂停完全交还上游。

## v0.3.10

**删除人物后自动暂停兼容**：内嵌 Control 0.2.1，修复删除离线玩家角色后服务器自动暂停状态的兼容性问题。发布门禁包含 Docker .NET 6 SDK 中的 Control 契约矩阵验证、只读真实 game-data 编译 Mod、后端全量测试、Docker integration、前端状态脚本和生产构建。Docker Desktop 真机验收覆盖删除后暂停、时钟稳定、重启和错误日志扫描。

## v0.3.9

**轮询资源泄漏与重复重启门禁**：修复 Panel 长时间运行时的轮询资源泄漏——邀请码查询改为只读 server 容器 `/tmp/invite-code.txt`，空值返回 `n/a`，不再启动交互式 `attach-cli`；邀请码与资源指标均增加按实例 5 秒缓存和 singleflight，多浏览器页面共享同一次 Docker exec/stats。活动重启 job 使用持久 operation payload 标识，重复提交返回 `409 restart_in_progress`，不会取消或替换原重启。前端在页面隐藏或关闭时停止轮询，恢复可见后再继续。

## v0.3.8

**删除离线存档人物**：管理员可以在服务器运行中删除当前活动存档里离线、非主机、已认领的玩家角色。删除前自动保存并创建整档保护备份，删除后发送游戏内通告，被删角色重新上线会自动中止连接。

## v0.3.7

**升级旧镜像安全清理**：面板升级成功后会自动清理被替换的旧镜像和历史积累的过期 tag 镜像，释放磁盘空间；当前镜像、其它项目镜像和正在被其它容器引用的镜像均受到保护。

## v0.3.6

**存档导入完整事务**：上传存档改为持久化导入事务（跨面板重启可恢复），支持跨平台存档导入时选择原主机处理方式（替换平台 ID 或保留身份），强制保存并验证存档写盘完整性，同名零修改自动拒绝。

## v0.3.5

**强制升级 Junimo 125 并完善模组农场**：新安装实例默认使用 JunimoServer 1.25；现有 1.21 实例不强制升级，管理员可通过面板自愿升级。模组农场创建功能进一步完善，修复兼容性问题。

## v0.3.4

**Mod 合包运行时完整性修复**：多 Mod ZIP 按每个 manifest 独立导入，上传摘要明确区分发现、实际安装、启用和跳过数量；修复 SMAPI 内置组件重复安装问题；服务器 compose 默认使用 OpenAL null 与 SDL dummy 音频后端，避免无声卡环境导致 Mod 加载中断；新增 SVE 旧存档兼容性警告。

## v0.3.3

**模组农场创建闭环**：新增离线模组农场目录扫描，支持安全 JSONC 解析、Content Patcher AdditionalFarms、中文 i18n 和受控图标；新增农场目录与依赖准备接口，新建游戏页展示模组农场并支持管理员一键准备前置 Mod；重构 custom-new-game 为持久化事务，保证原子写入和完整验证；Control Mod 升级到 0.2.0，输出事务绑定的 fresh runtime catalog。模组农场创建功能开关 `ENABLE_MODDED_FARM_CREATION` 默认关闭。

## v0.3.2

**组件运行时同步**：修复 JunimoServer DLL 在宿主机 mods 目录挂载后被遮蔽的问题，启动前自动从镜像同步官方 JunimoServer Mod，升级事务包含目标 Mod 提取与校验。

## v0.3.1

**配置自愈**：可信旧配置自动规范化修复，管理员在面板一键完成备份、规范化与验证，不再需要手改 `.env` 文件。

## v0.3.0

**模组农场目录**：离线扫描模组 Content Patcher 农场定义，展示已识别的模组农场卡片（名称、图标、依赖状态），支持一键准备前置 Mod；模组农场创建功能暂未开放，需后续运行时验证。

## v0.2.10

**组件升级代际修复**：修复组件升级任务在延迟竞态点击下的代际错乱问题，补齐发布门禁回归测试。

## v0.2.9

**Junimo 升级控制契约验收修复**：修复 Junimo server/auth 成对升级过程中控制契约验收的边界问题。

## v0.2.8

## v0.2.7

**运行组件升级修复**：修复 Junimo server/auth 运行组件升级过程中的边界问题，完善升级事务的稳定性。

## v0.2.6

**升级镜像候选回退**：修复组件升级时镜像候选拉取失败后的回退逻辑，确保升级流程在镜像源异常时能安全终止。

## v0.2.5

## v0.2.4

**连续升级修复**：修复历史成功升级覆盖后来发现的更高版本、旧 dry-run 误用于新版本目标的问题，旧成功不再永久锁住下一次升级。

## v0.2.3
