# 日常维护

面板日常运行期间的更新、备份、重启和管理操作。

## 更新面板

一键脚本部署的服务器：

```bash
cd ~ && bash run.sh update
```

如果更新后版本没变化（拉取失败或使用了旧缓存），强制重新拉取镜像并重建容器：

```bash
cd ~ && bash run.sh force-update
```

如果一键脚本本身也发布了新版本，先更新脚本再更新面板：

```bash
cd ~ && bash run.sh update-script
cd ~ && bash run.sh update
```

更新面板只会重建面板容器，不会删除 `~/.anxi-panel/data`；存档、Mod、数据库和备份都会继续保留。NAS 图形化部署的用户，在项目/应用栈里重新拉取镜像并重启容器即可，同样不会影响挂载的数据目录。

## 存档备份

"存档"页左下角"自动备份策略"可以配置：

- **游戏保存后自动更新最新备份**：勾选后，游戏内每次触发保存（含过夜自动保存），面板会自动刷新一份 `latest_<存档名>.zip`。
- **定时备份**：可指定每天几点（0-23 时）触发一次，生成/覆盖 `scheduled_<存档名>.zip`。
- **每日快照保留天数**：额外保留按日期命名的 `daily_<存档名>_<日期>.zip`，滑块可调保留天数（默认 3 天，最长 14 天）。

备份列表支持手动"恢复"（覆盖当前存档前会提示确认）和"删除"单个备份文件。重要节点（比如大量安装/删除 Mod 之前）建议手动确认一次备份是否为最新。

## 计划重启（定时维护窗口）

"服务器"页的"计划重启"可以设置每天固定的关闭时间和开启时间：

- 可设置关闭前提前几分钟通过游戏内聊天广播提醒在线玩家。
- 可选"关闭前自动备份当前存档"，关闭前先打包一份存档备份再停止服务器。
- 可选"有玩家在线时跳过本次关闭"，避免打断正在游玩的玩家。
- 到设定的开启时间会自动重新启动服务器；面板重启或长时间离线后，只在开启时间之后 5 分钟宽限窗口内补跑一次，不会补跑很久以前错过的时间点。

## Mod 管理

"模组"页支持：

- 手动上传 Mod ZIP。
- 搜索 Nexus Mods 并通过浏览器扩展一键安装（含依赖识别：会提示缺失或未启用的前置 Mod）。
- 配置管理员的 Nexus API Key 后，Premium 用户可以走直连安装，无需浏览器扩展。
- 查看每个 Mod 的依赖是否满足、当前存档是否启用，按需启用/禁用（联动同包和前置 Mod）。
- 删除、导出已安装的 Mod。

改动 Mod 后通常需要重启服务器才能生效，面板会在需要时提示"需要重启"。

## 面板用户与权限

"设置"页（管理员可见）可以创建、禁用、删除面板用户：

- 管理员可以管理所有用户；普通用户只能登录、登出、查看自己的信息，看不到用户管理、Docker 状态等管理入口。
- 最后一个启用中的管理员账号不能被禁用、删除或降级，当前登录的管理员也不能禁用/删除自己，避免误操作导致无人能管理面板。
- 删除用户默认是软删除（禁用），需要显式选择才会硬删除。

## 日志与诊断

- "任务日志"页可以查看安装、启动、停止、备份等所有后台任务的实时日志和历史记录。
- "诊断"页可以查看 Docker/Compose 版本、健康检查详情，并导出诊断包（管理员可见的"导出诊断包"按钮），内含版本信息、健康检查、任务日志、Compose 状态等，已自动脱敏密码/Token/邀请码。
- 定期查看审计日志（管理员操作记录），确认没有非预期的账号或配置变更。

## 安全维护清单

- `PANEL_SECRET` 使用足够长的随机字符串，且不要对外泄露。
- 管理员账号使用强密码，定期检查是否有陌生管理员账号。
- 只放行必要端口：面板 `TCP 8090`，游戏 `UDP 24642`/`UDP 27015`，按需 VNC `TCP 5800`；不要把 Junimo 内部 API `TCP 8080` 暴露到公网。
- 挂载 Docker Socket 等于给了面板容器较高的宿主机 Docker 控制权，不建议在多人共用的宿主机上部署，也不建议把面板本身长期暴露在公网且不设强密码。
- NAS 用户如果需要在外网管理面板，优先用 VPN、Tailscale、ZeroTier 或路由器自带 VPN，而不是直接把 `TCP 8090` 转发到公网。
