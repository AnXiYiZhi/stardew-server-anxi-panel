# 故障排查

按现象查找对应小节。如果本文没有覆盖你的问题，先按 [日常维护](maintenance.md) 里的"导出诊断包"收集信息，再反馈。

## 面板打不开 / 容器起不来

- 确认端口 `8090` 没有被其他程序占用；改用其他宿主机端口重新运行（一键脚本可在菜单里改端口，手动 `docker run` 时改 `-p 9090:8090`）。
- 确认启动命令挂载了 Docker Socket：`-v /var/run/docker.sock:/var/run/docker.sock`，否则面板无法控制游戏容器。
- NAS 图形界面如果不允许挂载 `/var/run/docker.sock`，面板无法正常工作，需换支持 Docker Socket 挂载的部署方式。

## 安装卡住不动 / 镜像拉取失败

- 面板对 JunimoServer 和 steam-auth 镜像都做了多镜像源候选兜底（阿里云 ACR、DaoCloud、GHCR、Docker Hub 等），单个源失败会自动尝试下一个，任务日志会显示 `(1/4)` 这样的候选进度。
- 如果所有候选都失败，通常是服务器整体出网络问题（DNS 解析失败或被墙）。面板内置了 DNS 自愈（系统解析失败会自动换用公共 DNS），但如果服务器完全不能访问公网 DNS 53 端口，这个自愈也无法生效，需要联系服务器提供商或检查防火墙。
- 家用 NAS 如果路由器 DNS 不稳定，也可能导致 Nexus 搜索、Docker Hub 版本检查间歇性失败，现象和上面类似。

## Steam 登录 / Steam Guard 一直卡住

- 安装页优先走非交互账号密码登录；如果 Steam 要求二次验证，页面会显示手机 App 批准提示或验证码输入框，按提示操作即可。
- 如果长时间没反应，先确认 `.env` 里的 `STEAM_SERVICE_IMAGE` 使用的是面板维护的 CN 修补镜像（`anxiyizhi/junimo-steam-service-cn` 系列），再查看任务日志里是否有连接重试信息。
- 反复登录失败时，可以先停止服务器，在"安装"页重新发起登录（不影响已下载的游戏文件）。

## 邀请码一直"获取中…" 或显示 n/a

- 邀请码走 Steam SDR / Galaxy P2P 中继，这条链路本身可能因为网络环境不稳定而卡住，不代表服务器没启动成功。
- 优先使用"玩家"页的**局域网邀请**（IP 直连）：面板默认开启 `Server.AllowIpConnections`，只要开放了 `UDP 24642`，好友可以直接用 IP 加入，不依赖邀请码。
- 如果始终没有邀请码也没法用 IP 直连，先确认服务器容器确实处于运行状态（"服务器"页显示"运行中"）。

## 好友/局域网玩家进不去游戏

- 最常见原因是只开放了 TCP 端口，漏了 UDP：Stardew 游戏端口 `UDP 24642` 和查询端口 `UDP 27015` 都必须放行，且必须是 UDP 而不是 TCP。
- 云服务器检查安全组规则；NAS 检查路由器端口转发是否把 `UDP 24642`/`UDP 27015` 转发到 NAS 的局域网 IP。
- 确认游戏内/面板里显示的服务器状态是"运行中"，而不是仍在安装或启动过程中。

## "当前激活存档"信息显示"未知"

- 刚新建存档时，游戏还没有触发过第一次自动存档（游戏内过夜/睡觉），面板暂时读不到农场主、日期等信息，属于正常的短暂中间状态，等游戏内过一天就会显示正常。
- 如果长时间（远超一次正常存档创建耗时）仍然显示"未知"，且局域网/邀请码都能正常进入游戏，说明游戏本身没问题，只是面板存档指针识别出现异常；更新面板到最新版本可以让这种情况自动修正，无需手动处理。

## 复制按钮（邀请码 / 局域网 IP）点击没反应

- 早期版本在通过 `http://IP:端口`（非 HTTPS）访问面板时，浏览器复制 API 不可用会导致复制按钮没有任何反应。更新面板到最新版本即可修复，复制按钮会自动降级使用兼容方案。

## Mod 相关：Nexus 搜索/安装失败

- Nexus 数字 ID 精确查询需要先在"模组"页配置管理员的 Nexus API Key；未配置时会明确提示缺少 Key，而不是报错 500。普通关键词搜索不需要配置 Key。
- 通过浏览器扩展一键安装 Mod 时，确认扩展已更新到面板提示的最新版本（扩展版本和面板不匹配可能识别不到 Nexus 页面的下载入口）。
- 批量安装多个 Mod（含前置）时，如果某个 Mod 之前已经安装过又被重复提交，面板会自动识别为"已安装，跳过重复导入"，不会算作失败。

## 存档启动失败

- 提示需要选择存档（`save_required`）：说明服务器已经安装好，但还没有可用存档，回到"存档"页新建或上传一个。
- 提示 VNC 端口不可用：说明宿主机上 VNC 端口（默认 `5800`）被占用，检查是否有其他程序占用该端口，或在设置里更换端口。

## 更新面板后版本没有变化

- 普通更新只会在本地已有对应镜像层时生效；如果拉取失败或使用了旧缓存，需要强制更新：

```bash
cd ~ && bash run.sh force-update
```

- 如果一键脚本本身也有更新，先更新脚本再更新面板：

```bash
cd ~ && bash run.sh update-script
cd ~ && bash run.sh update
```

## 还是没解决？

在"诊断"页导出诊断包（管理员可见），里面包含版本、健康检查、任务日志、Compose 状态等信息（已自动脱敏密码/Token），反馈问题时附带诊断包能大幅加快排查速度。
